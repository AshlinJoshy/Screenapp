1 — Supabase (PostgreSQL)

supabase.com · Free tier · No credit card needed

Step 1

Create an account & new project

Go to supabase.com → Sign up → click New Project

Fill in: Name = adscreen  |  Database Password = something strong (save it!)  |  Region = EU West (Ireland)

Save your database password now — you cannot retrieve it later.

Step 2

Wait for provisioning

Takes about 60–90 seconds. You'll see a green "Project is ready" banner.

Step 3

Get the connection string

Left sidebar → Project Settings → Database → scroll to "Connection string"

Click the URI tab. Copy the string. It looks like:

DATABASE_URL
postgresql://postgres:[YOUR-PASSWORD]@db.xxxxxxxxxxxx.supabase.co:5432/postgres

Replace [YOUR-PASSWORD] with the password you set in Step 1.

Use the "Transaction" mode connection string (port 6543) if you see connection limit errors.

Step 4

Copy to Railway

Go to Railway → API service → Variables → update DATABASE_URL with the string above.

2 — Upstash (Redis)

upstash.com · Free tier · 10K commands/day

Step 1

Create account & Redis database

Go to upstash.com → Sign up → Create Database

Fill in: Name = adscreen  |  Type = Regional  |  Region = EU-West-1 (Ireland)  |  TLS = Enabled

Step 2

Get the Redis URL

After creation, scroll down to the Connect section. Find the redis-cli command:

redis-cli command (shown on Upstash dashboard)
redis-cli --tls -u redis://default:YOUR_TOKEN@xxx.upstash.io:6379

Take that URL and change redis:// to rediss:// (double-s = TLS):

REDIS_URL to paste into Railway
rediss://default:YOUR_TOKEN@xxx.upstash.io:6379
Step 3

Copy to Railway

Railway → API service → Variables → update REDIS_URL

Do the same for the Worker service Variables — it also needs REDIS_URL.

5 — Run Database Migrations

Do this once after the API service first deploys successfully

Step 1

Wait for API to go green

In Railway, the API deployment shows a green ✓ when it's live. This takes 2–4 minutes on first deploy.

Step 2

Open the Railway shell

Click on the API service → top tabs → Shell

Step 3

Run these two commands in order

Push schema to Supabase
pnpm --filter @adscreen/db db:push
Create admin user
pnpm --filter @adscreen/api seed:admin

After this, your admin login is: admin@adscreen.io / Admin1234!

6 — Vercel (Next.js Dashboard)

vercel.com · Free forever for hobby projects

Step 1

Import the GitHub repo

Go to vercel.com → Add New Project → Import AshlinJoshy/Screenapp

Step 2

Configure the project

Before deploying, expand Root Directory and set it to:

Root Directory
apps/web

Framework should auto-detect as Next.js.

Step 3

Add environment variable

VariableValue
NEXT_PUBLIC_API_URLhttps://your-api.up.railway.app

Use the Railway domain from Section 3 Step 3 — no trailing slash.

Step 4

Deploy

Click Deploy. Takes ~2 minutes. You'll get a URL like screenapp-xxx.vercel.app.
